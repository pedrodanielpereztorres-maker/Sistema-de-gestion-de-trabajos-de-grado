### Qué es el SGT
Sistema de Gestión de Tesis y Pasantías del IUTEPI. Permite a los
administradores registrar estudiantes, asignar tutores y empresas, gestionar
tesis y generar reportes. Los estudiantes acceden desde cualquier dispositivo
para ver su información y perfil.

### Stack tecnológico
| Componente | Tecnología |
|---|---|
| Framework web | Reflex 0.9.1 (Python reactivo) |
| Base de datos | PostgreSQL — psycopg2 + pool SQLAlchemy |
| ORM | **NINGUNO** — SQL puro con cursores psycopg2 directos |
| Hashing | bcrypt directo (clase EncriptadorContrasena, límite 72 bytes) |
| Reportes | fpdf2 para PDF, csv nativo para exportaciones |
| Deploy | LAN local, acceso multiusuario simultáneo |

### Roles del sistema
| Rol | Dispositivo | Acceso |
|---|---|---|
| **administrador** | PC/Laptop (bloqueado en móvil por diseño) | Todo el sistema |
| **estudiante** | Multidispositivo (móvil, tablet, PC) | Solo su información |

### TAREA 3 🔴 CRÍTICA — `cargar_estadisticas_carrera()` tiene fuga de conexión

**Archivo:** `estado/estado_estudiante.py` — método `cargar_estadisticas_carrera()`  
**El problema exacto:**
```python
async def cargar_estadisticas_carrera(self) -> None:
    try:
        conn = obtener_conexion()          # Abre conexión
        if conn is None:
            ...
        with conn.cursor() as cursor:      # Usa cursor
            cursor.execute(...)
            resultados = cursor.fetchall()
        self.carreras_con_cantidad = [...]
    except Exception as e:
        logger.exception(...)
        self.carreras_con_cantidad = []
    # ← No hay conn.close() en ningún caso
    # La conexión queda abierta indefinidamente — fuga de conexiones
```
Sin `finally: conn.close()`, cada llamada a `cargar_estadisticas_carrera()`
(que ocurre al final de `cargar_estudiantes()`) deja una conexión abierta
en el pool. Con uso normal esto agota el pool de SQLAlchemy y hace que
el sistema deje de responder.

**Qué debes hacer:**
1. Capturar `conn` antes del try para poder usarla en finally:
   ```python
   conn = None
   try:
       conn = obtener_conexion()
       ...
   except Exception as e:
       logger.exception(...)
   finally:
       if conn:
           try:
               conn.close()
           except Exception:
               pass
   ```
2. Aplicar el mismo patrón que usa el resto del proyecto

Muéstrame el método `cargar_estadisticas_carrera()` completo y corregido.

### TAREA 5 🟠 IMPORTANTE — Sin rate limiting en el endpoint de login

**Archivo:** `estado/estado_autenticacion.py` — método `iniciar_sesion()`  
**El problema exacto:**
```python
def iniciar_sesion(self):
    correo_entrada = self.entrada_usuario.strip().lower()
    conn = obtener_conexion()
    # Se puede llamar este método infinitas veces sin límite
    # Un atacante puede hacer brute force sobre contraseñas con miles
    # de intentos por segundo sin ninguna penalización
```
El sistema no tiene ningún mecanismo para limitar intentos fallidos de login.
Un ataque de diccionario sobre una cuenta de administrador tardaría segundos.

**Qué debes hacer:**
1. Agregar una variable de estado para rastrear intentos:
   ```python
   intentos_fallidos: int = 0
   bloqueado_hasta: str = ""   # ISO datetime string
   ```
2. Al inicio de `iniciar_sesion()`, verificar si está bloqueado:
   ```python
   from datetime import datetime, timezone
   if self.bloqueado_hasta:
       limite = datetime.fromisoformat(self.bloqueado_hasta)
       if datetime.now(timezone.utc) < limite:
           segundos = int((limite - datetime.now(timezone.utc)).total_seconds())
           return rx.toast.error(f"Demasiados intentos. Espere {segundos} segundos.")
       else:
           self.bloqueado_hasta = ""
           self.intentos_fallidos = 0
   ```
3. Cuando la contraseña falla, incrementar el contador:
   ```python
   self.intentos_fallidos += 1
   if self.intentos_fallidos >= 5:
       from datetime import timedelta
       self.bloqueado_hasta = (datetime.now(timezone.utc) + timedelta(minutes=5)).isoformat()
       return rx.toast.error("Cuenta bloqueada por 5 minutos por múltiples intentos fallidos.")
   ```
4. Al login exitoso, resetear el contador: `self.intentos_fallidos = 0`

Muéstrame el método `iniciar_sesion()` completo con rate limiting.
